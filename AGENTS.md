# AGENTS.md — とむSYSTEM

このファイルはClaude Codeへの作業指示書です。

## プロジェクト概要

- `index.html` : フロントエンド（全アプリUI）
- `tomu_system_worker.js` : Cloudflare Worker（APIプロキシ・認証）
- `apps.json` : アプリ定義ファイル
- デプロイ: `deploy.ps1` または `deploy_wrangler.ps1`

## 現在のタスク: A-6〜A-11

以下を **index.html** を中心に実装すること。変更は最小限・既存コードスタイルに合わせること。

---

### A-6: ストリーミング対応

- Anthropic APIのストリーミングレスポンス（`stream: true`）を使い、回答を逐次表示する
- Workerside（`tomu_system_worker.js`）でSSEをフォワードし、フロント側でReadableStreamを受け取って表示

### A-7: XSSエスケープ強化

- `innerHTML` に直接ユーザー入力やAIレスポンスを挿入している箇所をすべて特定する
- テキストは `textContent` または適切なエスケープ関数を通じて挿入する
- Markdownレンダリングが必要な箇所はサニタイズ済みのものだけ `innerHTML` に渡す

### A-8: 会話履歴トリミング

- 会話履歴が長くなりすぎた場合にトークン超過エラーが起きないよう、古いメッセージから削除する
- 直近N件（例: 20件）または合計文字数上限（例: 40,000文字）を超えたら先頭から削除する

### A-9: 新しい相談ボタン

- 各アプリ画面に「新しい相談を始める」ボタンを追加する
- クリックで会話履歴をリセットし、初期状態に戻る
- 確認ダイアログは不要（即リセット）

### A-10: エラー時リトライボタン

- API呼び出しが失敗した場合、エラーメッセージと共に「再試行」ボタンを表示する
- 再試行ボタンをクリックすると同じリクエストを再送する

### A-11: アクセシビリティ改善

- ボタン・入力欄に適切な `aria-label` を付与する
- キーボード操作（Tab移動・Enter送信）が正しく動作するか確認・修正する
- コントラスト比が低い箇所があれば修正する

## 作業方針

1. タスクは A-6 → A-11 の順に実装する
2. 各タスク完了後にコミットする（例: `feat: A-6 ストリーミング対応`）
